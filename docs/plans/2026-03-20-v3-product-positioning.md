# OfferYou V3 Product Positioning

## One-line Definition

OfferYou is a mentor-guided career companion that helps people uncover real strengths from lived experience, navigate toward suitable directions, evaluate role fit, and output truth-based resumes that improve interview conversion.

## Core Promise

OfferYou does not begin with rewriting resumes.
It begins with understanding the person.

The product should help users answer five progressively sharper questions:

1. What have I actually done?
2. What does that reveal about my natural strengths and working style?
3. What kinds of roles or directions fit those strengths?
4. For a specific JD, how well do I match and where are the risks?
5. How should my true experience be expressed so that I can earn more qualified interviews?

The goal is not mass application volume.
The goal is to help people apply more honestly, more accurately, and more effectively to roles where they can actually perform well and grow.

## Product Flow

OfferYou's ideal user journey is:

`Mentor Companion -> Talent Excavation -> Career Navigation -> Role Matching -> Resume Output -> Higher Interview Rate`

### 1. Mentor Companion

The system should feel like a thoughtful mentor, not a black-box rewriter.

It should:
- guide the user step by step
- reduce ambiguity
- ask for confirmation where truth matters
- keep the user in control of what becomes durable profile memory

### 2. Talent Excavation

The first real asset is not the resume.
It is the user's truth base.

OfferYou should surface:
- repeated behaviors
- durable strengths
- problem-solving patterns
- work preferences grounded in evidence

This layer must always be rooted in first-hand, verifiable experience.

### 3. Career Navigation

Before narrowing to a single JD, the product should help the user understand:
- what kinds of work fit their strengths
- which directions are aligned
- which directions are possible but costly
- where the current evidence base is weak

This is what turns OfferYou from a resume tool into a career guidance product.

### 4. Role Matching

For a concrete role, OfferYou should answer:
- Do I match this role?
- Why might I fit?
- What evidence is missing?
- What are the main risks or stretch points?

This layer should feel like decision support, not keyword stuffing.

### 5. Resume Output

The final resume must not fabricate identity.

It should be:
- rooted in true experience
- reorganized for a specific target role
- clearer about strengths and evidence
- optimized to improve qualified interview conversion

## What Makes OfferYou Different

### Truth-first, not rewrite-first

Most tools start from wording.
OfferYou starts from reality.

### Mentor-guided, not black-box automation

The user should feel accompanied, not overwritten.

### Talent before tailoring

Most tools jump directly to a JD.
OfferYou first helps users understand themselves.

### Career navigation before resume optimization

The product should help users pursue better-fit work, not just prettier applications.

### Honest optimization

Every output resume should be a stronger expression of true capability.

## Core Principles

### Real experience is the source of truth
- no invention
- no silent embellishment
- no fake conversion hacks

### Human confirmation is mandatory
- facts require confirmation
- inferred insights remain separate from facts
- job-specific expression should not pollute the core profile

### The system should improve decisions, not just documents
- better self-understanding
- better role targeting
- better expression
- better interview conversion

## Implication For V3 UX

The user-facing interface should focus on:
- fit
- strengths
- gaps
- role readiness
- confidence in the judgment

Internal infrastructure like:
- Master facts
- source tracking
- reuse tracking

should mostly improve the trustworthiness of the judgment rather than dominate the visible interface.
