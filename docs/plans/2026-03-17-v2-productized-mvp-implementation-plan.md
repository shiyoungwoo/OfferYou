# OfferYou V2 Productized MVP Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Build the first runnable OfferYou Web MVP that supports Master creation, application drafts, suggestion review, snapshot generation, A4 preview, PDF export, and application records without polluting the Master.

**Architecture:** Start from a single-repo Next.js application with Prisma and a local storage adapter. Keep clear boundaries between ingestion, master, analysis, snapshot, and export so the MVP remains single-user in runtime but multi-user-ready in interface design. Use one canonical `ResumeDocument` JSON model rendered to HTML for both A4 preview and PDF export.

**Tech Stack:** Next.js App Router, TypeScript, React, Prisma, SQLite, Zod, Tailwind CSS, Vitest, Testing Library, Playwright, Prisma migrations, headless browser PDF rendering

---

## Assumptions

- The current repository has no runnable app yet.
- The implementation should live inside this repository rather than a separate new repo.
- The runtime may assume a default local user, but data access interfaces must already accept a `userId`.
- Initial file storage uses the local filesystem under an app-controlled storage directory.
- PDF generation will use headless browser rendering from HTML generated by the app.
